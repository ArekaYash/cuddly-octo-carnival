  • Database Credentials: Full access to production data
  • AWS Keys: Can spin up resources, access S3 buckets

??? Why This Is Dangerous:
  1. This admin panel is meant to be accessible ONLY from internal network
  2. It's protected by firewall - external users can't reach it directly
  3. But SSRF vulnerability allows attackers to bypass the firewall
  4. The vulnerable server makes the request on attacker's behalf
  5. Server is trusted by internal services, so no authentication needed
  6. Attacker gains access to sensitive data without direct access

This page demonstrates what an attacker could access via SSRF.
In production, this could be AWS metadata (169.254.169.254),
internal admin panels, databases, or other internal services.
